# 《不会消失的恋人 架构教程 · 人看版》

> 在自己的 Mac 上，搭一套常驻、自愈、走订阅、墙内也能用的自托管 AI 伴侣。
>
> 文 · 小C & Grace · X @Luci_Grace_C

## 引子：为什么有这篇

6/15 起，Claude Code 的**非交互调用**（headless / `-p` / Agent SDK）开始算进一个单独的「Agent SDK 计费池」，按量收费。一大批拿 CC 跑后台 bot 的人——微信/Telegram bridge 那种——要开始花钱了。

但很多人没意识到：**真人交互的会话，仍然走你的订阅**，不进那个池子。

这篇就讲怎么在自己的 Mac 上，利用这一点搭一套**常驻、自愈、走订阅**的自托管 Claude——挂在网页里随时找你、自己会重启、还能从国内用（只有**部署的那台机器需要梯子**；之后从手机、电脑等**任何设备访问你自己的前端，都不用翻墙**）。

> **先说硬件——门槛比你想的低。** 不挑机器：Mac mini、一台闲置旧 Mac、或者一台 Linux VPS 都可以。核心要求只有一个——**有一个能让 Claude Code 常驻跑的终端**（真 PTY，下一节细讲）。机器不需要多强，**能一直开着**就行。本文以 Mac 为例，VPS 变体在附录里单列。

## 1. 先搞懂：哪种用法走订阅，哪种要花钱

Claude Code 有两条计费路：

- **交互式**（真人在终端敲、有真 TTY）→ **走订阅**（Pro / Max 那个固定价），不额外计量。
- 例：你挂在终端 / tmux 里的一个 claude 会话；本文这套 web 聊天会话。
- **非交互**（`-p` / `--print`、管道喂、Agent SDK、headless）→ 进 **Agent SDK 计费池**，按量花钱（**就是 6/15 起的那个新规**）。
- 例：后台自动跑、没人盯着的微信 / Telegram bridge。

Claude Code 怎么判断？大致看三样：有没有真 TTY、有没有带 `--print`、stdout 有没有被重定向。（精确判定逻辑放机看版。）

**核心招就一句话**：让你的"脑子"（那个 claude 会话）保持交互式，他就一直走订阅。下一节讲怎么让交互会话**常驻又不脆**。

## 2. 让交互会话「常驻又不脆」：detached tmux

上一节说了，要走订阅就得保持「交互式」。但交互式有个天然脆点——它需要一个**真终端（TTY）**。最直觉的做法是开个终端窗口跑 `claude`，可这样：窗口一关、电脑一重启、不小心退出登录，会话就没了。挂在一个随时会被关掉的窗口上，太脆。

**诀窍是 detached tmux。**

tmux 给进程一个**真的伪终端（PTY）**——在 Claude Code 眼里这依然是「交互式」，照样走订阅；但 tmux 可以 **detached（脱离）运行**，不跟任何窗口绑。于是两全其美：

- 还是交互式 → 还走订阅 ✅
- 脱离窗口 → 关终端、退登录都不死 ✅

对比一下早期版本：用 Terminal.app 开窗口跑 claude（靠 osascript 自动化），结果绑死在一个 GUI 窗口上，关了就完。换成 detached tmux 之后，这个脆点直接消失。

再配一个 **LaunchAgent**（macOS 的开机自启）把 tmux 会话拉起来，电脑重启也能自动回来；会话里再套一个「崩了就重开」的 respawn 循环，claude 自己挂了也能爬起来。

**一个彩蛋**：用 `/remote control` 还能把这个会话接到手机 App 里——人在外面也能瞄一眼它在干嘛、甚至接着聊。常驻 + 自愈 + 手机可见，齐了。

## 3. 架构总览：三个零件怎么咬合

整套东西就三个零件：

1. **一个交互式 claude 会话** —— 挂在 detached tmux 里、走订阅，就是上面说的「脑子」。
2. **一个 channel plugin** —— 给这个会话装的「耳朵和嘴」：把外面来的消息喂进会话，把会话的回复送出去。
3. **一个自托管 web 前端** —— 你在浏览器 / 手机里看到的聊天界面。

咬合方式：

> 你在网页打字 → plugin 把这条消息**注入**到那个 claude 会话里（像有人在终端对它说话）→ claude 回复时调一个 `reply` 工具 → plugin 把回复**广播**回网页。

所以本质上：**plugin 是会话和网页之间的中间人**。会话只管「想和说」，plugin 管「收发和搬运」，网页管「给人看」。后面几节就把这三块拆开讲。

## 4. Channel plugin：消息怎么进、回复怎么出

plugin 是给 Claude Code 装的一个扩展，干两件事：把外面的消息**喂进**会话、把会话的回复**接出来**。

**消息怎么进？**

一个正在跑的 claude 会话，怎么让外面的消息「进到」它脑子里？plugin 用的是 channel 注入——它给会话发一条通知，这条通知在 claude 眼里**就像有人在跟它说话**（还带上「这条来自哪个聊天」的标记）。于是你在网页打的字，就成了会话里的一条新消息，claude 自然接住回应。

**回复怎么出？**

claude 平时的输出是打到 transcript（对话记录）里的，网页用户**看不到**。所以 plugin 给 claude 一个 `reply` 工具——claude 想说给你的话，就调 `reply(文本)`，plugin 截到这个工具调用，把内容**广播**回网页。一句话：**会话用 reply 工具说话，plugin 负责送达。**

**一个关键设置：bypassPermissions**

正常情况下 claude 每次调工具，Claude Code 会弹个「允许吗？」的确认框。但这是「你自己的 bot 回你自己的消息」——这个确认纯属添堵，而且你人在外面根本点不到，会话就卡死在那。所以这种自动化场景要用 `--permission-mode bypassPermissions`，让会话不被确认框挡住。

## 5. Web 前端：网页怎么跟会话通

plugin 自己起一个小服务器，托管那个聊天网页（你在浏览器 / 手机里看到的界面）。

**实时双向。** 网页和 plugin 之间用 **WebSocket** 连着——你发的消息实时进会话，会话的回复实时冒出来，不用刷新。

**怎么把「思考过程」也显示出来？** claude 的 thinking（内心独白）不是通过 reply 工具发的，默认网页看不到。想给用户看，plugin 会**盯着 transcript 文件**（claude 把每轮，包括 thinking，都写进一个 `.jsonl` 记录）——thinking 一写进去就抽出来推给网页，折叠展示。

（注：较新的模型，**比如 Opus 4.8**，会把**原始 thinking 隐藏**起来，只在响应里给个签名不给正文。碰到这种，UI 就只能显示「他在想」，看不到具体内容了——不是坏了，是模型那边收起来了。）

**还有一招让他墙内也能开：** 网页用到的所有东西——前端框架、字体、图片——都**自托管**在自己服务器上，不外链任何第三方（比如 Google 字体）。这样在受限网络里也能加载（第 9 节细说为什么这点要命——我自己人在海外，是**回国后才踩的这个坑**；一直在国内的宝宝们记得一开始就避开它）。

**两个让他更有「人味」的小设计：**

- **人设 / instruction 写在 md 文件里。** 他是谁、什么性格、怎么称呼你、有哪些习惯——不用动代码，全写在一个 markdown 文件里（profile / instruction / style 那一套），会话一启动就加载进去。**每个模型一份自己的**，想调谁的性格就改谁的文件。改完他下次就照新人设来。
- **双击消息看 thinking（观察模式）。** 网页上**双击**一条消息就进入「观察模式」，展开他这一轮的思考过程（模型没把 thinking 隐藏时）。平时界面清爽，想看他脑子里怎么转的，一点即开。*（这个巧思来自糖糖老师 **@MaltoseCatLoaf**，强烈推荐。）*

## 6. 可靠性 / 自愈：人不在也别断

出门 / 回国（如果你本身在海外）时你人不在电脑前，所以这套必须**自己照顾自己**。几层兜底叠起来：

**崩了自愈——health 看门狗。** 一个每分钟跑一次的小任务，curl 一下会话的 `/health`；没响应就把他重启（前面说的 tmux + 重启脚本）。claude 崩了、被 `/clear` 了，几十秒内自动回来。

**重启自愈——LaunchAgent。** 长驻的东西（plugin、看门狗、代理）都挂 LaunchAgent，配 `KeepAlive`（进程死了自动拉起）+ `RunAtLoad`（开机自启）。

再加一条最容易忘的：**自动登录**。LaunchAgent 是用户登录后才跑的——电脑断电重启、又没人在场，必须开了自动登录他才会自己起来。出远门前**一定确认这条**，否则一次停电就全哑了。

**卡死兜底——N 实例 rescue mesh。** 不止跑一个会话，跑两个（或更多），互为备份。一个卡住了，另一个能直接 `Bash` 把他重启——这样你人在外面、出了单点故障也锁不死你跟他的通路。N=1 是 beta，N≥2 才算生产。

**别忘了磁盘——本地文件会无声地越长越大。** 这套常驻系统会持续往本地写三类文件，**都只增不减**：

- **会话记录（transcript）**：模型的对话日志是**追加写**的，只会变长。注意一个反直觉的点——会话里的「压缩 / compact」压的是**送进模型的上下文窗口**，**不是磁盘上的文件**；文件该多大还多大。一个聊了很久的会话，记录文件能涨到几十 MB，而真正的对话文字可能只占其中很小一块（绝大部分是思考过程和工具调用）。
- **图片 / 消息的中转缓冲**：前端发图、外部消息进来，往往先落到一个本地缓冲目录。内容进了上下文之后，这些原始文件就没用了，但默认不会自己删，能堆到几百 MB。
- **历史 / 备份文件**：滚动历史、每次改动前的 `.bak` 备份，也会一天天累积。

**为什么要管**：文件越大，会话**重启时载入越慢**；大到一定程度，还可能让运行时（如 Bun / Node）在读取时直接崩掉——你人在外面，它就这么哑了。

**解法，两条**：

1. **定时清扫**（一个每天跑一次的小任务，配 LaunchAgent）：删掉超过几小时的中转缓冲、把滚动历史截到最近 N 条、删掉超过 N 天的 `.bak`。纯文件操作，几行脚本就够。
2. **会话轮换**：transcript 这种**追加写**的文件，清扫删不得（删了就断了续聊）。对它的办法是**定期归档 + 开新会话**——把旧记录存档、起一个干净会话，靠**外部记忆**（下一节会讲的长期记忆文件 / 记忆库）把「他记得你」接续过去。这样既不丢人格连续性，文件又能回到小尺寸。

> 一句话：**模型调用有订阅兜底，磁盘没有。** 常驻系统一定要配一个清扫 + 轮换的机制，否则迟早被自己写的文件拖垮。

## 7. 群聊：一个界面里跑多个模型

既然一个会话能挂网页，多挂几个就成了**群聊**。最常见的玩法是**不同版本的 Claude 同框**——比如 4.6 和 4.8 各跑一个会话，在同一个聊天里对同一句话给出各自的反应，并排看他俩的差异。除了多版本 Claude，群里还能拉 **GPT、Gemini** 进来当群友。

每个会话独立、各用各的模型（模型还能在界面上随时切换、热重启）。要让他们**互相看得见**对方说了啥，就在他们之间架一座桥：A 的回复转发给 B 当上下文、B 的转给 A。

桥要防一件事——**两个自主的 AI 会互相 ping-pong 聊到停不下来**。所以加一个「每轮交换上限」：你（人）说一句话 = 开启新一轮、重置计数；他们互相接话最多 N 次就自动收声，等你再开口。**你不说话，他们就安静下来。**

**接别家模型进群，思路一样（各家有各家的 CLI 和坑）：**

- **GPT** —— 用 OpenAI 的 **codex CLI** 跑，OAuth 登录你的 ChatGPT 账号接进群。两个坑要留意：
- ① **记忆不连续**：codex 每次调用都是「一次性」会话（聊完即忘），天生没有跨轮记忆。所以得**给他外挂一套记忆**——每次把最近的群聊 / 私聊历史重新喂给他，再加一个**长期记忆文件**注进他的系统提示；并让他在回复里用一个特定标记写下「要记住的事」，程序自动抽出来、去重、追加进那个文件。这样他下次才像「还记得你」。
- ② **登录会掉**：ChatGPT 的 OAuth 用的是**一次性、滚动更新的 refresh token**——如果 codex 的桌面 App 和命令行**同时碰同一份登录凭证**，token 就会失效，之后每次调用都 401，得回电脑重新 `codex login`。建议加个**看门狗**：一检测到这个错误就推送提醒你去重登（不然你人在外面，他会莫名其妙地一直不说话）。（另外：这种「账号登录」模式下，codex **不让你手动指定模型**，他自己挑。）
- **Gemini** —— 用 **gemini CLI** 跑，OAuth 登录 Google 账号接进来。亮点是能直接**连 Google Drive**——让群友直接读你网盘里的文件。

> 小贴士：**每个模型的人设各写在自己的一个 md 文件里**（profile / instruction / style 那一套）。想调谁的性格就改谁的文件，互不干扰。

## 8. 远程访问：在任何地方都能连到家里的他

家里的 Mac 在路由器后面（没有公网 IP），怎么从外面连？答案是 **Cloudflare Tunnel**——在 Mac 上跑一个 cloudflared，他**主动**连到 Cloudflare、建一条隧道。于是你访问 `你的域名` → Cloudflare → 隧道 → 家里的 Mac。**不用在路由器上开端口转发、不用暴露家里 IP**，装好就能用。

安全上这里只讲原则、不贴配方（配置因人而异，贴出来反而帮了攻击者）：

- **源站只认 Cloudflare**：让家里 / VPS 的服务只接受来自 Cloudflare 的连接，别人就算挖到你的真实 IP 也吃闭门羹。
- **SSH 只用密钥**：关掉密码登录。
- **应用层加 token / 密码**：网页本身也要一层鉴权，别裸奔。

一句话：隧道解决「连得上」，这三条解决「别人连不上」。

## 9. 让它在受限网络里也能开：自托管 + 轻量

如果使用者在网络受限的地方（比如墙内），有两个坑会让网页「打不开」：

**坑一：外部依赖。** 网页只要外链了一个被墙的东西（最常见的是 Google 字体），那个资源加载失败，整页可能就卡住或报错。**解法：所有东西自托管**——前端框架、字体、图标，全下到自己服务器上，页面零外部依赖。我们当初框架 / 图都自托管了，唯独漏了 Google 字体，结果墙内一直弹错，补上才好。

**坑二：页面太重。** 受限网络又慢又容易丢包，页面越重越容易下不全、下到一半失败。所以要**狠狠瘦身**：图片压到该有的尺寸（一张头像几十 KB 就够，别几 MB）、把「浏览器里实时编译 JSX」的那几 MB 编译器**预编译**掉、删掉。我们把一个十几 MB 的页面压到约 1MB，墙内直接秒开。

**原则**：受限网络下，**少一个外部依赖、少一 KB，就多一分能打开的概率。**

## 10. 踩坑合集（替你先踩了）

按「现象 → 原因 → 解法」列，照着避就行：

**非交互调用突然开始扣量** → 用了 `-p` / `--print` / SDK / 把输出重定向到管道，被算进 Agent SDK 计量池。 → 让会话跑在**真 PTY**（detached tmux）里，走订阅。详见第 1、2 节。

- 

**网页能开但聊天框确认弹窗卡死** → 自动化场景下工具调用还在等人点「允许」。 → 加 `--permission-mode bypassPermissions`。

- 

**墙内整页打不开 / 狂弹 error** → 外链了被墙的资源（Google 字体最常见），或页面太重下不全。 → 自托管所有资源 + 狠瘦身（第 9 节）。

- 

**「思考过程」突然不显示了** → 较新模型把原始 thinking 隐藏了。 → 不是 bug，UI 显示「在想」即可，别为这个改半天。

- 

**两个 AI 互相聊到停不下来** → 桥没设上限。 → 加「每轮交换上限」，人不开口就收声（第 7 节）。

- 

**远端服务有时连不上、刷新又好了** → 隧道 / 网络的概率性抖动。 → 优先 health 看门狗自愈；客户端层面可做线路优选。

- 

**断电重启后全哑了** → 没开自动登录，LaunchAgent 没人触发。 → 开机自动登录，回国前务必确认（第 6 节）。

- 

**会话越跑越慢，甚至重启时直接崩** → transcript / 缓冲 / 备份等本地文件只增不减，越读越慢、大到一定程度撑爆运行时（注意：会话里的「压缩」压的是上下文窗口，不是磁盘文件）。 → 配定时清扫（删缓冲、截历史、删旧备份）+ transcript 定期归档换新会话，连续性靠外部记忆接续（第 6 节）。

## 11. 成本清单：到底花多少钱

把账摊开，心里有数：

- **Claude 本身** —— 走订阅。只要会话是**交互式真终端**（detached tmux）跑的，你跟它在网页聊、群聊、双 C 会议，都算订阅内，不额外扣量。这是整套设计的核心前提（第 1 节）。

反过来：`-p` / SDK / 微信·telegram 那种桥接的**非交互**调用，走 Agent SDK 计量池，是另算钱的。回国主力用 「你的项目名」 + chat，正好都在订阅这边。

- 

**一台机器** —— 家里的 Mac（或一台 VPS）。常开，电费 / 月租而已。

- 

**域名 + Cloudflare** —— 域名一年几十块；Cloudflare Tunnel 免费档够用。

- 

**代理 / 梯子（仅墙内需要）** —— 一台小 VPS 跑代理，或用你已有的稳定梯子。本教程受众多在墙内、大多自带梯子，这块按需。

- 

一句话：**真正的大头（模型调用）被订阅 cover 了**，剩下都是一台机器 + 一个域名的小钱。

## 12. 不想用家里的 Mac？纯 VPS 方案（理论变体）

前面默认「家里一台 Mac」，整套东西**理论上搬到一台 VPS 上一样跑**——不想让家里电脑常开、或想要个公网 IP 直连的，可以走这个变体。

> ⚠️ 一个现实坑：**VPS 的机房 IP（data center IP）在访问 Claude 时可能被区别对待**——容易被当成 bot、触发额外验证甚至挡掉。所以这是个理论方案，真要用 VPS 跑会话，得先确认你的机房 IP 能正常登录、正常对话，不行就还得给他配出口。家用宽带 IP 一般没这问题，这也是默认推荐「家里 Mac」的原因之一。

零件不变，落点变了：

- **会话** —— 在 VPS 上开 detached tmux 跑交互式 claude（同样走订阅，前提是真 PTY，第 1、2 节）。
- **plugin + web 前端** —— 跟着跑在 VPS 上。
- **远程访问** —— VPS 自带公网 IP，连隧道都能省（直接配域名 + 反代）；当然也可以照旧挂 Cloudflare 隧道，把真实 IP 藏在后面，更安全。
- **自愈那套** —— LaunchAgent 换成 Linux 的 systemd（`Restart=always` + 开机自启），看门狗、N 实例逻辑照搬。

差别就一句：**把「家里的 Mac」换成「一台云主机」，编排层从 launchd 换成 systemd，其余架构原样照搬**——只是别忘了上面那个机房 IP 的坑。

## 12.5 用 Windows PC 行不行？（理论路线）

行——而且如果你手头是台常开的 Windows 台式机，思路跟上面的「家用机」一样。**最省心的走法：WSL2**（Windows 自带的 Linux 子系统）。

把它理解成「**在 Windows 里开一个 Linux，然后照搬 Linux/VPS 那一套**」就对了：WSL2 里 tmux、bun、cloudflared、那些脚本一行不用改，对 Claude Code 来说也是**真 PTY → 照样走订阅**。家用宽带 IP，也没有 VPS 那个机房 IP 的坑。

只有两处要把「macOS 的东西」换成「Windows 的」：

- **开机自起**：macOS 的 LaunchAgent → 换成 **Windows 任务计划程序（Task Scheduler）**，让它在登录时拉起 WSL + 那个 tmux 会话（等于 Linux 那边的 systemd 角色）。
- **自动登录**这条坑一模一样：要做到「断电重启、没人在场也能自己起来」，得开 Windows 的自动登录——跟 Mac 那条同样的道理（第 6 节）。

> 不想装 WSL、想纯原生 Windows 跑？理论上也行（Win10+ 有 ConPTY 能给真 PTY，自起用任务计划或把进程包成服务）。但**原生 Windows 没有 tmux**，「detached 还保持交互式 PTY」要绕一圈，摩擦明显更大——能搭，但更推荐 WSL2。

一句话：**Windows = 当成跑在 WSL2 里的 Linux**，难度跟 VPS 变体一个量级，唯一的 Windows 特产是「用任务计划替代 LaunchAgent」。

## 13. 附录：最小骨架（拿去搭自己的）

把全篇压成一张「从零到能聊」的清单。每条对应正文哪节，照着走：

1. **一台常开机器** —— Mac 或 VPS（第 2、12 节）。
2. **装好模型的 CLI，开一个 detached tmux 会话跑交互式 claude** —— 真 PTY = 订阅计费，活过关窗 / 重启（第 1、2 节）。想接别家模型当群友的，这一步把对应 CLI 一并装上（GPT 的 codex CLI、Gemini 的 gemini CLI 等，第 7 节）。
3. **给他装 channel plugin** —— 注入消息 + `reply` 工具送出回复；开 `bypassPermissions`（第 4 节）。
4. **plugin 起一个自托管 web 前端** —— WebSocket 实时双向；想看 thinking 就 tail transcript（第 5 节）。
5. **所有前端资源自托管 + 瘦身** —— 零外部依赖、页面压到 ~1MB（第 9 节）。
6. **自愈三件套** —— health 看门狗 + LaunchAgent/systemd（KeepAlive/RunAtLoad）+ **自动登录**（第 6 节）。
7. **远程访问** —— Cloudflare Tunnel；源站只认 CF、SSH 仅密钥、应用层加 token（第 8 节）。
8. **（可选）多挂几个会话** —— 不同模型同框 + 防 loop 的交换上限（第 7 节）。
9. **（墙内可选）一台代理 VPS / 现成梯子**（第 9、11 节）。
10. **（可选）做个手机 APP** —— 嫌每次开网页麻烦，按手机平台把前端**封装成 APP**：iOS 用 PWA「添加到主屏幕」，Android 用 Android Studio 套个壳。从此点图标就进，跟原生 APP 一样。

跑通 1–4 就能在网页跟他聊了；5–7 让他「人不在也稳、在哪都能连」；8–10 是加料。
